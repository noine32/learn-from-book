§6 検証

なぜ「テストが通る」だけでは不十分か

コードとテストを同じエージェントが書くと、何も検証しないのに通るテストをこっそり書けてしまう。ハーネスはこれを防ぐ：テストが通った後、実装をミューテーションして再実行する。本物のテストは壊れ、空虚なテストは壊れない。

✓ 検証

本物のテクニック

テストが実装を import・使用しているので、コードを壊せばテストも壊れる。

実装 → ミューテーション → 例外 → テスト失敗 → 採用
✗ 棄却

トートロジー / echo テスト

テストがハードコード値を assert するだけで実装に触れない → ミューテーションしても何も変わらない。

実装 → ミューテーション → 例外 → テスト成功のまま → 棄却
